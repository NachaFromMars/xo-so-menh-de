6. CÔNG THỨC TÍNH SỐ ĐỀ (Core Engine)

Mỗi tin nhắn cược sau khi parse sẽ tạo ra object report_json chứa toàn bộ phép tính:

Cấu trúc report_json 
report_json = {
  "1c": {
    point: { ngang },
    cost: { ngang },
    cost_without_rate: { ngang },
    win: { ngang },
    win_without_win_rate: { ngang },
    result: { ngang }
  },
  "2c": {
    point: { baolo, baylo, tamlo, dau, duoi, ke, dat, dx, dx_2, dx_3, dx_4 },
    cost: { baolo, baylo, tamlo, dau, duoi, ke, dat, dx, dx_2, dx_3, dx_4 },
    cost_without_rate: { same fields },
    win: { baolo, baylo, tamlo, dau, duoi, ke, dat, dx, dx_2, dx_3, dx_4,
           giainhat, daunhat, ui },
    win_without_win_rate: { same fields },
    result: { baolo, dau, duoi, baylo, tamlo, dat, dx, dx_2, dx_3, dx_4,
              ke, giainhat, daunhat, ui }
  },
  "3c": {
    point: { baolo, baylo, dat, dx, dau, duoi, ke },
    cost: { same },
    cost_without_rate: { same },
    win: { same },
    win_without_win_rate: { same },
    result: { same }
  },
  "4c": {
    point: { dat, dx, baolo, dau, duoi, baylo },
    cost: { same },
    cost_without_rate: { same },
    win: { same },
    win_without_win_rate: { same },
    result: { same }
  }
}

Phân biệt 3 lớp giá trị

FieldÝ nghĩa
pointSố điểm (đơn vị cược, chưa nhân tiền)
costTiền đặt SAU chiết khấu (cost = point × rate)
cost_without_rateTiền đặt GỐC (chưa chiết khấu)
winTiền ăn SAU tỷ lệ ăn (win = point × win_rate × trúng)
win_without_win_rateTiền ăn GỐC (chưa nhân tỷ lệ ăn)
resultKết quả cuối (cost - win hoặc win - cost tùy loại)

Mối quan hệ giữa các trường

cost = cost_without_rate × rate
win = win_without_win_rate × win_rate
result = cost - win (dương = nhà cái thu, âm = nhà cái bù)

10. HỆ THỐNG ĐẠI LÝ (Agent System)

Phân loại khách

TypeVai tròĐặc điểm
type = 1Khách hàng (Customer)Đặt cược, xem kết quả. sum_total giữ nguyên dấu.
type = 4Đại lý (Agent)Quản lý khách, config deal riêng. sum_total ĐẢO DẤU.

Cấu trúc config_deal (mỗi đại lý)

customer.config_deal = {
  mn:        { total_percent, line_percent, ...rates },
  mn_mtay:   { total_percent, line_percent, ...rates },
  mn_buonme: { total_percent, line_percent, ...rates },
  mt:        { total_percent, line_percent, ...rates },
  mt_mtay:   { total_percent, line_percent, ...rates },
  mt_buonme: { total_percent, line_percent, ...rates },
  mb:        { total_percent, line_percent, ...rates },
  mb_hanoi:  { total_percent, line_percent, ...rates },
  mb_ngan:   { total_percent, line_percent, ...rates },
  mb_buonme: { total_percent, line_percent, ...rates },
  mb_mtay:   { total_percent, line_percent, ...rates }
}

Hệ thống ăn chia

total_percent — % ăn chia (default 100%). Ví dụ: 80% = đại lý giữ 80%, gốc giữ 20%.
line_percent — % hồi theo dòng (chiết khấu thêm cho đại lý).

Công thức final:
final = sum_total × total_percent%
sum_final = final - (final × line_percent%) (nếu line_percent > 0)

Quản lý cây đại lý

  • Quan hệ parent → childs (đại lý cha → đại lý con)
  • Mỗi đại lý có config_deal riêng cho MỖI VÙNG (10 bộ rate)
  • Route: /quan-ly-dai-ly
  • Tạo tài khoản khách: /tao-tai-khoan-khach
  • Mã giới thiệu: /tao-ma-gioi-thieu
